## Context

Production LLM deployments suffer from a recurring problem: average latency is acceptable, but the 99th percentile spikes when clusters mix heterogeneous hardware. This project was born from the need to serve inference with a latency target under 40 ms in an environment where GPUs from three different generations coexist.

## Architecture

The system is organized in three decoupled layers:

- **Ingest Layer** — normalizes and groups requests into dynamic micro-batches according to traffic pressure.
- **Compute Node** — a pool of workers with hardware-aware routing: each request is assigned to the node whose historical latency curve minimizes the expected p99.
- **Inference API** — output layer with circuit breakers and deadline-aware scheduling queues.

```python
def route(request: Request, nodes: list[Node]) -> Node:
    """Tail-latency-aware routing per node."""
    candidates = [n for n in nodes if n.healthy]
    return min(candidates, key=lambda n: n.p99_estimate(request.token_count))
```

## Technical decisions

The most debated decision was abandoning static batching. A fixed batch maximizes throughput but punishes the p99 during off-peak hours. Dynamic batching with an adaptive window reduced tail latency by **38%** without sacrificing more than 4% of aggregate throughput.

The retraining pipeline runs on Airflow with experiment tracking in MLflow; every routing-policy change is validated against a 72-hour replay of real traffic before being promoted.
